# Product Spec: 股票分析进度可视化

## Context

**现状：**
- 股票分析（特别是全面分析模式）耗时较长（30-60 秒）
- 用户只看到一个静态的 loading spinner："正在深度分析 AAPL..."
- 无法感知分析进度，不知道系统在做什么
- 用户体验差，容易误以为系统卡死

**用户痛点：**
1. 等待时间长，焦虑感强
2. 不知道分析进行到哪一步
3. 无法判断是否出现问题
4. 缺乏对 AI 分析过程的信任感

## Product Goals

**核心目标：** 让用户清楚地看到分析进度和关键步骤，减少等待焦虑，增强对系统的信任

**成功指标：**
- 用户能看到每个分析阶段的状态
- 用户能理解系统正在做什么
- 减少"分析卡住"的用户反馈
- 提升分析过程的透明度

## User Stories

### Story 1: 实时进度展示
**作为** 用户
**我想要** 看到分析的实时进度
**以便** 知道系统正在工作，还需要等多久

**验收标准：**
- 显示当前正在执行的分析步骤
- 显示已完成的步骤（打勾）
- 显示等待中的步骤（灰色）
- 显示失败的步骤（红色 X）

### Story 2: 关键信息预览
**作为** 用户
**我想要** 在分析过程中看到关键发现
**以便** 提前了解分析结果，减少等待感

**验收标准：**
- 每个步骤完成后显示 1-2 句关键发现
- 例如："护城河分析完成：发现强大的品牌护城河"
- 例如："财务分析完成：ROE 持续 > 20%"

### Story 3: 分析阶段说明
**作为** 用户
**我想要** 理解每个分析步骤的含义
**以便** 知道系统在做什么，建立信任

**验收标准：**
- 每个步骤有清晰的名称
- 可选：hover 显示该步骤的详细说明
- 步骤顺序符合逻辑（并行步骤同时显示）

## Design Proposal

### UI 布局

```
┌─────────────────────────────────────────────────────────┐
│  正在分析 AAPL                                          │
│                                                         │
│  ✓ 护城河分析                                           │
│    发现强大的品牌护城河和生态系统优势                    │
│                                                         │
│  ✓ 管理层分析                                           │
│    管理层执行力强，资本配置优秀                          │
│                                                         │
│  ⏳ 财务分析                                            │
│     正在分析财务指标...                                  │
│                                                         │
│  ⏸ 估值分析                                             │
│     等待财务分析完成                                     │
│                                                         │
│  ⏸ 综合决策                                             │
│     等待所有分析完成                                     │
│                                                         │
│  [取消分析]                                             │
└─────────────────────────────────────────────────────────┘
```

### 状态图标

- ✓ 已完成（绿色）
- ⏳ 进行中（蓝色，带动画）
- ⏸ 等待中（灰色）
- ✗ 失败（红色）

### 分析步骤

**全面分析模式（5 步）：**
1. 护城河分析（Moat）
2. 管理层分析（Management）
3. 财务分析（Financial）
4. 估值分析（Valuation）
5. 综合决策（Decision）

**单项分析模式（1 步）：**
- 只显示对应的单个步骤

### 关键信息提取规则

每个步骤完成后，从 AI 返回的报告中提取 1-2 句关键发现：

**护城河分析：**
- 提取护城河类型和强度
- 例如："品牌护城河强大，网络效应显著"

**管理层分析：**
- 提取管理层评价
- 例如："CEO 执行力强，资本配置优秀"

**财务分析：**
- 提取核心财务指标
- 例如："ROE 持续 > 20%，现金流充沛"

**估值分析：**
- 提取估值结论
- 例如："当前估值合理，安全边际充足"

**综合决策：**
- 提取投资建议
- 例如："建议买入，目标价 $200"

## Technical Approach

### Phase 1: 基础进度展示

**后端改动：**
1. API 支持 Server-Sent Events (SSE) 或 WebSocket
2. Orchestrator 通过 `on_progress` 回调发送进度事件
3. 事件格式：
   ```json
   {
     "type": "progress",
     "step": "moat",
     "status": "running|completed|failed",
     "message": "正在分析护城河...",
     "summary": "发现强大的品牌护城河" // 可选
   }
   ```

**前端改动：**
1. 创建 `AnalysisProgress.tsx` 组件
2. 监听 SSE 事件，更新进度状态
3. 显示步骤列表和状态图标
4. 替换现有的 `<Spinner>` 组件

### Phase 2: 关键信息提取

**后端改动：**
1. 每个 Agent 完成后，使用简单规则或 LLM 提取 1-2 句摘要
2. 在 progress 事件中包含 `summary` 字段

**前端改动：**
1. 显示每个步骤的摘要信息
2. 动画效果：摘要文字逐字显示

### Phase 3: 增强交互

**功能增强：**
1. 取消分析按钮
2. 步骤 hover 显示详细说明
3. 失败步骤显示错误信息和重试按钮
4. 进度百分比显示

## Out of Scope (V1)

- 实时流式输出完整报告内容
- 分析时间预估
- 历史分析进度回放
- 自定义分析步骤顺序

## Success Metrics

**定量指标：**
- 用户等待过程中的跳出率下降 > 30%
- "分析卡住"反馈减少 > 50%
- 分析完成率提升 > 20%

**定性指标：**
- 用户反馈：等待体验改善
- 用户对 AI 分析过程的信任度提升

## Implementation Priority

**P0 (Must Have):**
- 基础进度展示（步骤列表 + 状态图标）
- SSE 或 WebSocket 实时通信
- 替换现有 Spinner

**P1 (Should Have):**
- 关键信息摘要提取
- 取消分析功能
- 失败重试

**P2 (Nice to Have):**
- 步骤详细说明
- 进度百分比
- 动画效果优化
