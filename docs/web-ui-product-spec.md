# MoatLab Web UI — 产品功能设计

## 一、产品定位

面向个人价值投资者的 Web 工具，基于巴菲特/段永平投资哲学，提供 **AI 驱动的股票分析** 和 **持仓管理** 能力。核心价值：让价值投资的研究流程系统化、可追溯、可回顾。

### 目标用户

- 个人美股投资者，认同价值投资理念
- 希望用系统化方法替代主观判断
- 需要对持仓进行定期回顾和纪律约束

### 核心使用场景

| 场景 | 描述 |
|------|------|
| 发现标的 | 通过筛选条件快速过滤出符合价值投资标准的股票 |
| 深度研究 | 对单只股票进行多维度 AI 分析（护城河、管理层、财务、估值） |
| 买卖决策 | 基于 AI 综合分析结果，决定是否建仓/加仓/卖出 |
| 持仓管理 | 记录交易、跟踪盈亏、查看持仓分布 |
| 定期回顾 | AI 检查每个持仓的投资逻辑是否仍然成立 |

---

## 二、信息架构

```
MoatLab
├── 仪表盘 (/)                    # 全局总览，快捷入口
├── 持仓管理 (/portfolio)          # 交易记录、持仓跟踪
│   ├── 当前持仓 (默认 Tab)
│   ├── 交易历史
│   └── 持仓回顾
├── 股票分析 (/analysis)           # AI 深度分析
└── 股票筛选 (/screener)           # 条件筛选
```

## 三、页面功能规格

### 3.1 仪表盘 (`/`)

**目的**：一眼掌握投资全局，快速进入高频操作。

#### 功能模块

| 模块 | 内容 | 数据来源 |
|------|------|---------|
| 持仓汇总卡片 | 总市值、总成本、总回报(金额+百分比)、持仓数量 | `GET /api/portfolio` |
| 持仓分布饼图 | 按市值占比展示各持仓权重 | `GET /api/portfolio` |
| 赢家/输家 | 盈亏排行前 3 名（绿色/红色高亮） | `GET /api/portfolio/performance` |
| 快捷入口 | "分析股票"搜索框 + "买入"按钮 | 跳转到对应页面 |

#### 交互说明

- 页面加载时自动拉取持仓数据
- 搜索框输入 ticker 按回车 → 跳转至 `/analysis?ticker=AAPL`
- 饼图 hover 显示持仓详情（股数、市值、占比）
- 点击饼图某个持仓 → 跳转至该 ticker 的分析页

---

### 3.2 持仓管理 (`/portfolio`)

**目的**：记录交易、跟踪所有持仓的实时盈亏。

#### Tab 1: 当前持仓（默认视图）

**持仓表格**

| 列 | 说明 |
|-----|------|
| 股票代码 | ticker，可点击跳转分析页 |
| 持仓股数 | 当前持有数量 |
| 买入均价 | 加权平均成本 |
| 现价 | 实时市场价格 |
| 市值 | 现价 × 股数 |
| 盈亏金额 | 市值 - 成本，绿涨红跌 |
| 盈亏比例 | 百分比，绿涨红跌 |
| 操作 | "加仓" / "减仓" 按钮 |

**底部汇总行**：总成本、总市值、总回报

**买入/卖出表单（侧边抽屉 Drawer）**

点击页面顶部"买入"按钮 或 表格中"加仓/减仓"按钮，弹出侧边抽屉：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 股票代码 | 文本输入 | 是 | 自动大写，若从表格进入则预填 |
| 操作类型 | 买入/卖出切换 | 是 | 若从"加仓"进入默认买入，"减仓"默认卖出 |
| 股数 | 数字输入 | 是 | 正整数 |
| 价格 | 数字输入 | 是 | 每股价格(USD) |
| 交易日期 | 日期选择器 | 否 | 默认今天 |
| 备注 | 文本区域 | 否 | 交易理由 |

- 提交后表格自动刷新
- 卖出时校验：不能超过当前持仓股数

#### Tab 2: 交易历史

**交易记录表格**

| 列 | 说明 |
|-----|------|
| 日期 | 交易日期 |
| 操作 | 买入(绿) / 卖出(黄) |
| 股票代码 | ticker |
| 股数 | 交易数量 |
| 价格 | 每股成交价 |
| 备注 | 交易备注 |

- 支持按 ticker 筛选（下拉选择，选项来自历史交易的去重 ticker）
- 默认显示最近 50 条

#### Tab 3: 持仓回顾

- 点击"开始回顾"按钮，分析当前持仓
- 展示 Markdown 格式的回顾报告
- 加载中显示骨架屏/Spinner + "正在分析您的持仓..."
- 报告内容包括：每个持仓的投资逻辑检查、集中度风险、再平衡建议

### 3.3 股票分析 (`/analysis`)

**目的**：对单只股票进行 AI 驱动的价值投资分析。

#### 搜索区域

- 输入框：placeholder "输入股票代码，如 AAPL"
- 分析模式选择（Radio 或 SegmentedControl）：
  - **全面分析**（默认）— 护城河 + 管理层 + 财务 + 估值 + 决策
  - **护城河分析** — 品牌、网络效应、转换成本、成本优势
  - **管理层分析** — 诚信度、资本配置、股东导向
  - **财务分析** — FCF、ROE、负债率、利润率
  - **估值分析** — DCF、Owner Earnings、安全边际

- 点击"开始分析"按钮发起请求

#### 分析结果区域

- **加载状态**：全屏 Spinner + 提示文案 "AI 正在深度分析 {ticker}..."（全面分析可能需要 30-60 秒）
- **全面分析模式**：结果按章节展示，每个维度一个可折叠/展开的 Section：
  - 护城河分析
  - 管理层分析
  - 财务分析
  - 估值分析
  - 投资决策（默认展开，其余默认折叠）
- **单项分析模式**：直接展示 Markdown 报告
- Markdown 渲染支持：标题、表格、列表、加粗、代码块
- 报告底部显示"基于此分析买入"按钮 → 打开买入抽屉，ticker 预填

#### 历史分析记录（可选，后续迭代）

- 本地缓存最近 10 次分析结果
- 可快速查看历史分析，无需重新调用 API

---

### 3.4 股票筛选 (`/screener`)

**目的**：通过量化条件筛选符合价值投资标准的股票候选。

#### 筛选条件表单

| 字段 | 类型 | 说明 |
|------|------|------|
| 最低 ROE | 数字输入(%) | 如 15 表示 ROE >= 15% |
| 最高负债率 | 数字输入 | 如 0.5 表示 D/E <= 0.5 |
| 最低毛利率 | 数字输入(%) | 如 40 表示毛利率 >= 40% |
| 最高 PE | 数字输入 | 如 25 表示 PE <= 25 |
| 行业 | 下拉选择 | Technology / Healthcare / Consumer 等 |

- 至少填写一个条件才能提交
- "开始筛选"按钮

#### 筛选结果区域

- 加载状态：Spinner + "AI 正在筛选..."
- 结果为 Markdown 格式，直接渲染
- 结果中的 ticker 可点击 → 跳转至 `/analysis?ticker=XXX` 进行深度分析

## 四、全局设计规范

### 4.1 布局

```
┌──────────────────────────────────────────────────┐
│  LOGO(MoatLab)          [仪表盘] [持仓] [分析] [筛选]  │
├──────────────────────────────────────────────────┤
│                                                  │
│                    页面内容区                      │
│                                                  │
│                                                  │
└──────────────────────────────────────────────────┘
```

- 顶部导航栏固定，包含 Logo + 四个页面入口
- 当前页面对应的导航项高亮
- 内容区居中，最大宽度 1280px
- 响应式：移动端导航栏折叠为汉堡菜单

### 4.2 视觉风格

- 配色：深色主题为主（投资工具常见），浅色主题可后续支持
- 数据颜色语义：盈利/正向 = 绿色，亏损/负向 = 红色
- 字体：系统默认等宽字体用于数字，无衬线字体用于文本
- 间距和圆角：TailwindCSS 默认值，保持一致性

### 4.3 交互通用规则

| 场景 | 处理方式 |
|------|---------|
| API 加载中 | Spinner + 描述性文案 |
| API 成功 | Toast 通知（绿色，3 秒自动消失） |
| API 失败 | Toast 通知（红色）+ 显示错误详情 |
| 空数据 | 友好提示 + 引导操作（如"暂无持仓，去买入第一只股票"） |
| 长时间分析 | 进度提示，不阻塞其他操作 |

### 4.4 状态管理原则

- 持仓数据全局共享（仪表盘和持仓页都需要）
- 分析结果页面级状态，切换 ticker 时清空
- 筛选条件保留在页面状态，刷新后丢失可接受

## 五、API 依赖清单

现有后端 API 已满足所有核心功能，无需新增端点：

| 功能 | API | 状态 |
|------|-----|------|
| 持仓列表 | `GET /api/portfolio` | 已有 |
| 买入 | `POST /api/portfolio/buy` | 已有 |
| 卖出 | `POST /api/portfolio/sell` | 已有 |
| 组合绩效 | `GET /api/portfolio/performance` | 已有 |
| 交易历史 | `GET /api/portfolio/history` | 已有 |
| 股票分析 | `POST /api/analyze/{ticker}` | 已有 |
| 股票筛选 | `POST /api/screen` | 已有 |

### 需要新增的 API

| 功能 | 建议端点 | 说明 |
|------|---------|------|
| 持仓回顾 | `POST /api/portfolio/review` | 当前仅 CLI 有此功能，Web API 未暴露 |

## 六、实施优先级

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 持仓管理（表格 + 买入/卖出） | 高频操作，替代 CLI 的核心价值 |
| P0 | 股票分析（搜索 + 报告渲染） | 产品核心能力 |
| P1 | 仪表盘（汇总 + 饼图） | 提供全局视图，用户打开首页的第一印象 |
| P1 | 股票筛选（条件 + 结果） | 完整投资流程闭环 |
| P1 | 交易历史 | 完整持仓管理 |
| P2 | 持仓回顾 | 需新增 API，独立迭代 |
| P2 | 深色/浅色主题切换 | 用户体验优化 |
| P2 | 分析历史缓存 | 避免重复调用，节省 API 成本 |
